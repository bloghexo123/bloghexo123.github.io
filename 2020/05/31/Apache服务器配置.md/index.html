Apache服务器配置

老梁 · 2020-5-31 · 次阅读

注意：此笔记由我同学所写，笔记转载而来，再次感谢此同学——

注意：网页文件名不要瞎起，index.html

检查挂没挂载 （*）

cd /mnt/cdrom

配置 yum源 [root@local ~]# vim /etc/yum.repos.d/dvd.repo（*）

[dvd]  
 name=dvd 
 baseurl=file:///mnt/cdrom 
 gpgcheck=0  
 enabled=1

注意： （*）

 1、文件中“=”左右两侧不要留空格。 
 2、baseurl：指定YUM源的访问路径。这一行非常重要。 
 指向Web服务器：baseurl=http://… 
 指向本地某个目录： baseurl=file://…  
 指向FTP服务器： baseurl=ftp://… 

全部设置成仅主机：然后子网ip填192.168.10.0

启动：

systemctl start httpd

设置自启动:(*)

 systemctl enable httpd

网卡设置路径

/etc/sysconfig/network-scripts/ifcfg-ens**

复制文件

cp ifcfg-ens33 ifcfg-ens**

编辑

vim ifcfg-ens**

设置:

BOOTPROTO=static  #static设置 
IPADDR=192.168.10.100    #ip地址,一个100，一个101
NETMASK=255.255.255.0 #子网掩码
GATEWAY=192.168.10.2 #网关

重启:

ifdown ens** 
ifup ens**

挂载

mount /dev/sr0 /mnt/cdrom

安装apache

yum install httpd -y

检查是否安装

rpm -qa httpd

查找配置文件：

rpm -qc httpd

启动重启apache

systemctl restart httpd

开启

systemctl start httpd

设置自启动

systemctl enable httpd

开放80端口

firewall-cmd --zone=public --add-port=80/tcp --permanent 

查看放行列表

firewall-cmd –-list-all

放行http会话

firewall-cmd --permanent --add-service=http

必须重新加载(重点)：

firewall-cmd --reload

查看当前SELinux的应用模式，是强制、执行还是停用。

setenforce 0

临时生效可以直接用setenforce 0
1 启用， 0 告警，不启用

路径创建

mkdir /web/ipone 
mkdir /web/iptwo

填充内容

echo "11111111">/web/ipone/index.html 
echo "222222222">/web/iptwo/index.html

路径切换:

cd /etc/httpd/conf.d

内容设置

vim vhost.conf

设置基于IP地址为192.168.8.100的虚拟主机


<virtualhost 192.168.10.100> 
DocumentRoot /web/ipone 
</virtualhost>

设置基于IP地址为192.168.8.101的虚拟主机

<Virtualhost 192.168.10.101> 
DocumentRoot /web/iptwo 
</Virtualhost>

复制粘贴在一个文件里
修改权限

vim  /etc/httpd/conf/httpd.conf

注释掉

##DocumentRoot "/var/www/html"

复制粘贴框子里的

内容改为

<Directory "/web/iptwo">
AllowOverride None
# Allow open access:
Require all granted
</Directory>
<Directory "/web/ipone">
AllowOverride None
# Allow open access:
Require all granted
</Directory>

重启

systemctl restart httpd

2.基于域名：要建立基于域名的虚拟主机，DNS服务器中应建立多个主机资源记录（也可在客户端的hosts文件中添加域名记录），使它们解析到同一个IP地址。例如：
www.kidko.com.               IN    A    192.168.8.100

www.organic.com.                IN    A    192.168.8.100

（1）修改httpd.conf文件。添加目录权限内容如下。

    <Directory "/var/www">
        AllowOverride None
        Require all granted
    </Directory>

（2）修改httpd.conf文件。添加目录权限内容如下(之前以做过设置):

    <Directory "/var/www">
        AllowOverride None
        Require all granted
    </Directory>

（3）修改/etc/httpd/conf.d/vhost.conf文件。该文件的内容如下（原来内容清空）。

    <Virtualhost 192.168.8.100>
      DocumentRoot /share/web/kidko 
      ServerName  www.kidko.com
    </Virtualhost>
    
    <Virtualhost 192.168.8.100>
      DocumentRoot /share/web/organic 
      ServerName  www.organic.com
</Virtualhost>

（4）在客户端/etc/hosts文件中增加域名解析记录。

（5）SELnux设置为允许，让防火墙放行httpd服务，重启httpd服务（见前面操作）。

（6）在客户端浏览器中可以看到http:// www.kidko.com和http:// www.organic.com两个网站的浏览效果。

基于端口的虚拟主机

（1）修改/etc/httpd/conf/httpd.conf文件。该文件的修改内容如下。

Listen 8088   192.168.80.100                 
Listen 8089   192.168.80.100
<Directory "/share/web">
    AllowOverride None
    Require all granted
</Directory>

（2）修改/etc/httpd/conf.d/vhost.conf文件。该文件的内容如下（原来内容清空）。

<Virtualhost 192.168.8.100:8088>
         DocumentRoot  /share/web/kidko 
</Virtualhost>
<Virtualhost 192.168.8.100:8089>
        DocumentRoot   /share/web/organic  
</Virtualhost>

极为重要：

[root@local ~]# firewall-cmd --permanent --zone=public --add-port=8089/tcp
[root@local ~]# firewall-cmd --permanent --zone=public --add-port=8088/tcp
[root@local ~]# firewall-cmd --reload
